1.  Project Overview
AcadOS is a simulated operating system for a university's shared compute server. It understands academic context — who submitted a job, what type it is, and when it's due — and makes smarter CPU, memory, and I/O decisions based on that knowledge.
This document is the complete task-by-task execution plan. Each person owns one module. Every module has defined inputs and outputs so all five pieces connect seamlessly at the end.
CRITICAL RULE: Every person exposes their module through the interfaces defined in Section 3. Do not change function names or signatures after Day 1 morning. Shakti's shared.py is the contract everyone builds against.

5.  Task-by-Task Execution Plan
Tasks are ordered by dependency — earlier tasks must be completed before later ones can start. Tasks within the same number (e.g. Task 2) can run in parallel.
Task 1 is Shakti alone. Everyone else starts Task 2 only after shared.py is pushed to GitHub.
TASK 1  —  FOUNDATION  |  shared.py + PCB Struct  |  Must finish before anyone else starts
Shakti  —  Foundation Engineer
📋 TASK 1  —  shared.py + PCB in C + State Machine
• ⚙️  Set up GitHub repo — create folders: acados/, c_src/, tests/, outputs/. Invite all 4 teammates.
• 📦  Create shared.py — define PCB dataclass with all fields: pid, user_id, role, job_type, deadline, cpu_budget_ns, urgency_score, abuse_flag, state, cpu_used, pages, resources_held, disk_requests.
• 🏷️  Define all enums — Role (STUDENT=1, RESEARCHER=2, FACULTY=3), JobType (PRACTICE=1, RESEARCH=2, EXAM=3, EVALUATION=4), State (NEW through THROTTLED = 8 states).
• 🧮  Write compute_urgency() method on PCB — formula: urgency = (role_weight × jobtype_weight) / max(0.01, deadline - time.time()). Role weights: S=1, R=2, F=3. JobType weights: PRACTICE=1, RESEARCH=2, EXAM=4, EVAL=5.
• 🔄  Write transition_state(pcb, new_state) — validates legal transitions (e.g. RUNNING→BLOCKED is valid, RUNNING→NEW is not). Raise ValueError on illegal transition.
• 🔧  Write pcb.c — C struct mirroring the Python PCB. Compile: gcc -shared -fPIC -o c_src/pcb.so c_src/pcb.c
• 🐍  Write ctypes bridge in shared.py — load_pcb_from_c() loads the .so and maps C struct fields to Python PCB object.
• ✍️  Write Makefile — 'make' compiles both .c files. 'make test' runs pytest. 'make clean' removes .so files.
• 📤  Push shared.py to main branch and message the group — everyone can now start their module.
✅  shared.py imported successfully by all 4 teammates with no errors. All enums, PCB fields, and transition_state() confirmed working.

Sachi  —  Scheduler Engineer
📋 TASK 2  —  Three-Tier CPU Scheduler
• ⚙️  Install: pip install matplotlib. Create scheduler.py, import PCB, Role, JobType, State from shared.
• 📋  Build three queues — Tier1: PriorityQueue sorted by urgency_score descending (negate for max-heap). Tier2: PriorityQueue with aging counter. Tier3: collections.deque for Round Robin.
• 📥  Write submit_job(pcb) — call pcb.compute_urgency(), check job_type, place in correct tier queue. EXAM/EVALUATION → Tier1. RESEARCH → Tier2. PRACTICE → Tier3.
• ⏱️  Write scheduler_tick(tick_number) — check Tier1 first (EDF: pop lowest deadline). If Tier1 empty, check Tier2. If both empty, pop from Tier3 deque. Return (running_pcb, preempted_pcb or None). Call transition_state() on both.
• 🔄  Implement context switch — save_context(pcb): store {'PC': tick, 'REG': tick*2} into a dict on the PCB. load_context(pcb): restore the dict. Call both inside scheduler_tick().
• 📈  Implement aging for Tier2 — every 5 ticks, add 0.1 to urgency_score of all waiting Tier2 jobs so they eventually get scheduled.
• 🧪  Test with dummy data — create 3 EXAM + 3 PRACTICE + 2 RESEARCH PCBs. Run 30 ticks. Print which PID runs each tick. Confirm EXAM always gets CPU first.
• 📊  Write plot_gantt(timeline_list) — takes [(pid, job_type_str, tick)] list. Plot coloured horizontal bars per PID. EXAM=red, RESEARCH=orange, PRACTICE=green. Save to outputs/gantt.png.
✅  Scheduler runs 30+ ticks with mixed jobs. Gantt chart saved to outputs/gantt.png showing EXAM jobs dominating CPU time.
Ragini  —  Memory Engineer
📋 TASK 2  —  Paging + Deadline-Aware Page Replacement
• ⚙️  Create memory.py, import PCB, Role, JobType, State from shared.
• 🏗️  Write MemoryManager class — total_frames=32 (simulated RAM). free_frames = list(range(32)). page_table = {} (dict: pid → {virtual_page: frame}). lru_tracker = {} (dict: (pid, vpage) → last_access_tick). swap_space = {} (dict: (pid, vpage) → 'data').
• 📄  Write allocate_pages(pcb, num_pages) — pop num_pages frames from free_frames, build page_table[pcb.pid] mapping. If not enough frames, call _deadline_replace() first.
• 🔍  Write TLB — self.tlb = {} with max size 8. On access_page(pcb, vpage): check TLB first (hit → return frame). Miss → check page_table. Not in page_table → call _page_fault_handler().
• ⚡  Write _deadline_replace() — victim selection: collect all pages from PRACTICE PCBs first, then RESEARCH, then EXAM. Within same job_type, pick the page with the oldest lru_tracker entry (LRU). Evict it to swap_space. Update page_table and TLB.
• 🔁  Write _page_fault_handler(pcb, vpage) — load page from swap_space back into a free frame. Update page_table[pcb.pid][vpage] and TLB. Increment a fault counter per job_type.
• 🆓  Write free_pages(pcb) — remove all entries for pcb.pid from page_table and TLB. Add frames back to free_frames.
• 🧪  Test — fill all 32 frames with 4 PRACTICE + 1 EXAM PCB. Access EXAM page. Confirm only PRACTICE pages evicted. Check page_table entries are correct.
• 📊  Write plot_page_faults(acados_faults, lru_faults) — grouped bar chart: x-axis = job types, y-axis = fault count, two bars per group (AcadOS vs LRU). Save to outputs/page_faults.png.
✅  Deadline-aware replacement confirmed: EXAM pages stay in RAM even when memory is full. Page fault comparison graph saved to outputs/page_faults.png.
Nikhil  —  Deadlock Engineer
📋 TASK 2  —  Banker's Algorithm + Synchronization
• ⚙️  Create deadlock.py, import PCB, Role, JobType from shared. Also create c_src/banker.c.
• 🏗️  Write ResourceManager class — available = {'CPU': 4, 'MEM_BLOCK': 8}. allocation = {} (pid → resources dict). max_need = {} (pid → max resources dict). Register a process with register(pcb, max_dict).
• 🔧  Write banker.c — function is_safe_state(available[], allocation[][], need[][], n_processes, n_resources) → returns 1 if safe, 0 if not. Implements standard Banker's safety algorithm. Compile: gcc -shared -fPIC -o c_src/banker.so c_src/banker.c
• 📡  Write request_resources(pcb, request_dict) — compute need = max_need[pid] - allocation[pid]. Call is_safe_state via ctypes with hypothetical new allocation. If safe: update allocation + available and return True. If not: return False (job waits).
• 🆓  Write release_resources(pcb) — add pcb's allocation back to available. Remove from allocation dict.
• 🧪  Test Banker's — set up 3 processes that would deadlock if all granted. Confirm request_resources returns False for the unsafe one.
• ⚠️  Write deadlock_recover(all_pcbs) — sort processes by job_type (PRACTICE first, EXAM last). Terminate first process: call release_resources, set state to EXIT. Repeat until safe state restored. Return list of terminated PIDs.
• 🔒  Write AcadosSemaphore class — readers_count=0, mutex=threading.Lock(), write_lock=threading.Lock(). db_read(pid): acquire reader lock (allow multiple readers). db_write(pid, data): acquire write_lock exclusively. This implements Readers-Writers.
• 🧪  Test sync — spawn 5 reader threads + 1 writer thread simultaneously. Confirm no race conditions and data stays consistent.
✅  Banker's Algorithm rejects unsafe resource requests. Deadlock recovery terminates PRACTICE first, never EXAM. Readers-Writers semaphore tested with concurrent threads.
Sanat  —  I/O & Systems Engineer
📋 TASK 2  —  Disk Scheduling + Anti-Abuse Daemon
• ⚙️  Install: pip install rich matplotlib. Create io_manager.py, import PCB, Role, JobType, State from shared. Also import transition_state from shared.
• 💿  Write DiskScheduler class — current_head=0. Write cscan(requests, head): implement C-SCAN algorithm (scan in one direction, jump to start, scan again). Write sstf(requests, head): implement SSTF (pick closest cylinder to current head each time).
• 🧪  Test disk schedulers — use requests=[95, 180, 34, 119, 11, 123, 62, 64, 66], head=50. Print service order for both. Verify C-SCAN and SSTF give different orderings.
• 📊  Write plot_disk_seeks(cscan_order, sstf_order) — line chart: x-axis = service sequence, y-axis = cylinder number. Two lines (C-SCAN in blue, SSTF in orange). Save to outputs/disk_seeks.png.
• 🗄️  Write sqlite3 setup — create_db() creates acados.db with table: job_log(pid, user_id, role, job_type, cpu_used, start_time, end_time, missed_deadline). Write log_job(pcb): insert a row using pcb fields.
• 🔔  Write FileSystem class — QUOTAS = {Role.STUDENT: 10, Role.RESEARCHER: 100, Role.FACULTY: 500}. create_job_dir(pcb) adds entry to a dict. check_quota(pcb, usage_gb) raises QuotaError if over limit.
• 🚨  Write AbuseMonitor(process_table, tick_interval=5) as threading.Thread subclass — in run(): loop every tick_interval seconds. For each pcb in process_table: if pcb.cpu_used > 2 * pcb.cpu_budget_ns: set pcb.abuse_flag=True, halve pcb.cpu_budget_ns, call transition_state(pcb, State.THROTTLED) from shared.
• 🖥️  Wire up rich display — use rich.table.Table to print a live process table every 10 ticks: columns = PID, Role, JobType, State, CPU Used, Abuse Flag. Colour THROTTLED rows in red.
• 🧪  Test daemon — create a PCB with cpu_used = 3x cpu_budget_ns. Start AbuseMonitor. After 5 ticks confirm pcb.state == State.THROTTLED and pcb.abuse_flag == True.
✅  C-SCAN and SSTF produce correct orderings. Disk seek graph saved. AbuseMonitor correctly throttles abusive processes. sqlite3 DB logs jobs. rich table displays live process state.

main.py — Integration Script
# main.py — run with: python main.py
import os, time
import os; os.makedirs('outputs', exist_ok=True)
from shared import PCB, Role, JobType, State, transition_state
from scheduler import submit_job, scheduler_tick, plot_gantt
from memory import allocate_pages, access_page, free_pages, plot_page_faults
from deadlock import request_resources, release_resources, deadlock_recover, create_db
from io_manager import log_job, AbuseMonitor, plot_disk_seeks, cscan, sstf
create_db()   # initialise sqlite3
# Create jobs
jobs = [
  PCB(pid=1, user_id=101, role=Role.STUDENT,    job_type=JobType.EXAM,     deadline=time.time()+300),
  PCB(pid=2, user_id=102, role=Role.STUDENT,    job_type=JobType.PRACTICE, deadline=time.time()+3600),
  PCB(pid=3, user_id=201, role=Role.RESEARCHER, job_type=JobType.RESEARCH,  deadline=time.time()+1800),
  PCB(pid=4, user_id=103, role=Role.STUDENT,    job_type=JobType.PRACTICE, deadline=time.time()+7200),
  PCB(pid=5, user_id=301, role=Role.FACULTY,    job_type=JobType.EVALUATION,deadline=time.time()+600),
]
# Allocate + request + submit
for job in jobs:
    allocate_pages(job, num_pages=4)
    request_resources(job, {'CPU': 1, 'MEM_BLOCK': 2})
    submit_job(job)
# Start abuse monitor as background thread
monitor = AbuseMonitor(process_table=jobs, tick_interval=5)
monitor.start()
# Run 50 ticks
timeline = []
for tick in range(50):
    running, preempted = scheduler_tick(tick)
    if running:
        access_page(running, virtual_page=tick % 4)
        running.cpu_used += 1
    timeline.append((running.pid if running else -1, tick))
# Cleanup + log + graphs
monitor.stop()
for job in jobs:
    release_resources(job)
    free_pages(job)
    log_job(job)
plot_gantt(timeline)
plot_disk_seeks(cscan([95,180,34,119,11,123,62,64,66], 50),
               sstf([95,180,34,119,11,123,62,64,66], 50))
